# 개발 프로세스 (Issue → PRD → 작업 → PR)

## 1) Issue 작성

- 제목: `[타입] 요약` (`feat`, `fix`, `chore`, `docs` 등)
- 필수: 문제/배경, 완료 기준(AC), 스크린샷·로그(있다면)

## 2) PRD 문서 (`docs/tasks/NNN-slug.md`)

- 파일명: 3자리 번호 + kebab 슬러그 (예: `001-init.md`)
- 섹션: 목표, 배경, 범위, 비범위, AC, 산출물, 리스크/롤백
- 링크: 관련 이슈 번호, 디자인/백엔드 API 레퍼런스

## 3) 브랜치

- 패턴: `<type>/<issue>-<slug>` 예) `feat/12-retro-list`

## 4) 작업/배치

- 위치: `.cursorrules`의 FSD 규칙을 따른다.
- 커밋: 한 논리 단위로 쪼개고 Conventional Commits 적용.

## 5) PR

- 템플릿 필수. 포함: 개요, 변경사항, 체크리스트, 테스트 방법, 스크린샷, 영향/리스크, 관련 이슈·PRD 링크.
- 제목 예: `feat: 회고 목록 조회 추가 (#12)`

## 6) 머지 후

- PRD에 실제 산출물/결과 기록(체크박스), 릴리즈 노트 반영 시 링크 추가.
